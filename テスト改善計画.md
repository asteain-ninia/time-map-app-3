## フェーズ 3 計画: 複合シナリオテスト

### 方針

単体テストでは検出できない**操作の連鎖**や**状態遷移の組み合わせ**をテストする。Vitest上で複数UseCase/Serviceを組み合わせた結合テストとして実装する。

### 3.1 シナリオ一覧

| # | シナリオ | 関連モジュール | 推定テスト数 |
| --- | --- | --- | --- |
| S1 | 頂点移動 → ポリゴン検証 → 自己交差検出 → Undo → 検証解除確認 | VertexEditUseCase, PolygonValidationService, UndoRedoManager | 3〜4 |
| S2 | 地物追加 → 共有頂点化 → 一方の頂点移動 → 連動確認 | AddFeatureUseCase, SharedVertexService, VertexEditUseCase | 3〜4 |
| S3 | ポリゴン作成 → 穴追加 → 飛び地追加 → リング階層検証 | AddFeatureUseCase, RingEditService, PolygonValidationService | 3〜4 |
| S4 | 2地物作成 → 同レイヤー重複 → 競合検出 → 結合 → 競合解消確認 | AddFeatureUseCase, ConflictDetectionService, MergeService | 3〜4 |
| S5 | ナイフ分割 → 2地物に分離 → それぞれの形状整合性確認 | KnifeService, SplitFeatureCommand, PolygonValidationService | 2〜3 |
| S6 | 錨編集（時間範囲変更）→ 競合発生 → 解決 → コミット | PrepareFeatureAnchorEditUseCase, ResolveConflicts, CommitUseCase | 3〜4 |
| S7 | 複数Undo → Redo → 中間状態でのデータ整合性確認 | UndoRedoManager, 各Command | 2〜3 |
| S8 | プロジェクト保存 → 読込 → 全データの一致確認（ラウンドトリップ） | SaveLoadUseCase, JSONSerializer | 2〜3 |

## テスト追加の方法（全フェーズ共通）

- `it.todo('説明')` で宣言し、テストスイートを green のまま維持
- バグを発見したテスト → `it.skip` にして `現状.md` に不具合登録
- 正常に通るテスト → そのままコミット
