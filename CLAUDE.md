## Claude の役割

- 主な担当: **コードレビュー・コミット・ドキュメント更新**
- 実装は主に Codex が担当する。Claude はレビュー側として機能する
- 広域の探索・調査は **Explore サブエージェント**に委譲し、主コンテキストを節約する

## 作業開始時のチェック

- 実装/修正着手前に [開発ガイド.md](開発ガイド.md) 5.1.2 の「AI作業開始ゲート」に従い、§6 の関連教訓を節番号・見出し・本文付きで引用し、今回の適用方針を明示する（詳細: `memory/feedback_read_dev_guide.md`）
- 不具合修正時は [実装済み.md](実装済み.md) の近い履歴と [開発ガイド.md](開発ガイド.md) §6.7 の対応表を確認する。根本原因が§6にない場合は、履歴追記だけで済ませず§6へ教訓化する
- 運用メモリ索引 `memory/MEMORY.md` を確認する（メモリと開発ガイドは役割が別物）

## コミット運用

- 粒度は機能単位で細かく（詳細: `memory/feedback_commit_granularity.md`）
- Co-Authored-By はコードを書いた側に合わせる（詳細: `memory/feedback_co_author_convention.md`）
- コミットごとに 現状.md / 実装済み.md も更新する（詳細: `memory/feedback_update_genjou_md.md`）
- `git commit` は allowlist に入れない方針。メッセージ内容をユーザーが都度確認する

## 環境メモ（一時）

- Windows + Git Bash 環境。Read/Edit/Write のパス解決に失敗した場合は、**小文字ドライブ + 正スラッシュ**（例: `e:/WORKSPACE/time-map-app-3/...`）で試す。`cd` でプロジェクトルートに戻る操作は原則不要（cwd は常にプロジェクトルート）
- このセクションは再発しなくなったら削除する
