## 1. プロジェクト概要

### 1.1 基本情報

- **プロジェクト名**: 地物時空変遷編纂機
- **目的**:
    - 架空世界の歴史を、ユーザーが時間軸に沿って網羅的に視覚化・追跡できるツールを提供する。
    - 対象となる世界の歴史的出来事や勢力の変遷を地図上で管理することで、ユーザーは歴史的な進展や地理的変化をより実際に即した形で管理できる。

- **使用環境**:
    - スタンドアロンのデスクトップアプリケーションとして動作。
    - **プラットフォーム**: Electron。
    - 通信なしでオフライン環境でも利用可能。

### 1.2 システム全体像

- **クライアント**: Electron.jsベースのデスクトップアプリケーション
- **データ保存**: ローカルファイルシステム（JSON形式）
- **ユーザーインターフェース**:
    - メインマップ表示
    - 編集ツールバー
    - タイムライン（スライダー）
    - プロパティパネル
    - 情報一覧サイドバー

## 2. 機能要件

### 2.1 地図表示機能

- **ベースマップ表示**:
    - SVG形式の地図情報の表示をサポート。
    - 地図の図法は正距円筒図法であることを前提とする。
    - ユーザーが地図の赤道長（km単位）を入力できる。
    - ベースマップのSVG要素はマウスイベントを一切受け付けず（`pointer-events: none`）、テキスト選択も不可とする。地物操作やパン操作がベースマップ要素に阻害されてはならない。
    - 横方向無限スクロールを実現し、地図の東端・西端をまたぐ領域を表示・編集可能。
    - ズームイン・ズームアウト、パン操作をサポート。
    - **パン操作**:
        - **表示モード**では、地図上をマウスの**左ボタンドラッグ**または**中ボタンドラッグ**でパン操作を行える。
        - **追加モード**および**編集モード**では、地図上をマウスの**中ボタンドラッグ**でパン操作を行う。
    - **ズーム操作**: 全てのモードにおいて、マウスホイールの回転によってカーソル位置を中心にズームイン・ズームアウトを行う。
    - 世界地図全体から都市内地図レベルまでの広範囲かつスムーズなズームを実現。
    - ズームレベルは設定にて一定範囲に制限可能。
        - デフォルトのズーム範囲: 最小1倍、最大50倍
        - ユーザーによる設定可能範囲: 最小0.1倍～最大10000倍
    - アプリケーション起動時の初期表示は、中心経度0度、中心緯度0度、地図全体が表示されるズームレベルとする。

- **グリッド線と軸の表示:**
    - 地図上に、10度間隔の緯線・経線グリッドを表示する。
    - 赤道、本初子午線、180度経線を強調表示する（色、太さを変える）。
    - グリッド表示のON/OFFを切り替えるボタン。
    - グリッドの間隔を設定可能（5度、10度、15度、30度）。
    - グリッドの色と透明度を設定可能。
    - 緯度・経度のラベルは、ズームレベルに関わらず画面上で一定のフォントサイズ（読みやすさを考慮した範囲内）で表示される。

- **測量モード**
    - ユーザーが地図画像上で2点クリックすると、その2点間の距離を計算して表示するモード。
    - 正距円筒図法に基づく距離と、大円距離（球面上の最短距離）の両方を計算して表示する。
    - 世界端をまたぐ測定線は折り返して表示し、分断されないようにする。
    - 測定モードを解除しても測定線は残り、明示的にクリアするまで表示される。
    - 大円距離に基づく線を描画する。
        - このとき、確実に最短経路を通るようにする。
        - 描画された測量線は一時的なもので、保存されない。
        - 複数の測量線を同時に表示可能。
        - 測量線のクリアボタンを提供。
    - ユーザーが地図画像上をクリックした地点の緯度経度を表示する。
        - 緯度経度は度分秒形式と十進形式の両方で表示。
        - クリックした地点に一時的なマーカーを表示。
    - ユーザーがボタン操作で、地図の表示経度を左右に10度ずつシフトできる。これにより、本初子午線中心の地図と太平洋中心の地図などを切り替えられる。
        - 「経度シフト」スライダーまたは±10度ボタンで操作。
        - 現在の中心経度を表示。

- **地理情報の表示**:
    - 点の大きさ・線の太さおよび表示される名前ラベルは、ズームレベルに関わらず画面上での見た目の太さ/サイズが一定になるように調整される。
    - 以下の情報を表示可能：
        - **点情報**: 都市、出来事など
            - 位置（単一頂点に紐づく）
            - プロパティ（名前、説明、年代など）
            - 視覚的表現（アイコン、色、サイズなど）
        - **線情報**: 道路、交易路、鉄道など
            - 複数頂点のパス
            - プロパティ（名前、説明、年代など）
            - 視覚的表現（線種、色、太さなど）
            - 長さを計算し、表示する（km単位）。
            - 国境線を表すことはない。
        - **面情報**: 国家、領域（無政府地帯を含む）など
            - 閉じたパス（複数頂点）で構成されるリング構造で形状を定義。
            - 任意の数の穴（内部に除外される領域）や飛び地（同一地物に属する別の独立した領域）、およびそれらのネスト構造に対応する。
            - プロパティ（名前、説明、年代など）
            - 視覚的表現（塗りつぶし色、境界線など）
            - 面積を計算し、表示する（km²単位）。
            - 飛び地は分離して描画し、人工的な線で繋がないようにする。
            - すべてのレベルの面情報（基礎自治体を含む）で飛び地をサポート。
            - 選択やハイライトなど一時的な視覚効果でも、穴領域は常に透過とし、飛び地は独立した領域として本土同様に表示する。
                - 恒常表示と異なる輪郭・塗り分けを行わないこと。

- **穴あき領域・飛び地の編集**:
    - 面情報に穴（内部に除外される領域）や飛び地（同一面情報に属する別の外周）を追加・編集・削除できる。
    - **穴/飛び地追加ツール**:
        - 編集モードで面情報を選択し、「穴/飛び地追加」ツールを選択。
        - 地図上をクリックして新しい領域（穴または飛び地）の頂点を追加（最低3点）。
        - 最初のクリックが選択中の面情報の**内部**（既存の穴を除く）であれば「穴作成モード」、選択中の面情報の**外部または既存の穴の内部**であれば「飛び地作成モード」となる。モードは領域作成完了またはキャンセルまで維持される。
            - 穴作成モード中は、クリックした地点を含む外周リングの**内部**（かつ既存の穴の外部）にしか頂点を追加できない。
            - 飛び地作成モード中は、選択中のポリゴンの**外部または既存の穴の内部**にしか頂点を追加できない。
            - 境界線上のクリックはいずれのモードでも無効とする。
        - 確定ボタンまたはダブルクリックで領域の追加を完了。この際、後述の「2.2.3 編集時の時間的・空間的整合性維持プロセス」プロセスがトリガーされる。
        - 追加された領域は、リング一覧に新しいリングとして追加された状態を歴史の錨として記録する。穴の場合は「穴リング」、飛び地の場合は「領土リング」として扱う。
        - この「穴/飛び地追加ツール」は、選択された単一の面情報の形状を編集するものであり、この操作によって新しい独立した面情報が生成されることはない。
            - 面情報から一部をくり抜いて新しい独立した面情報を作成する場合は、後述の「2.1 分裂と割譲/合邦機能」内の「穴あけ分割」機能を使用する。
    - **穴/飛び地の編集操作**:
        - 既存の穴または飛び地の頂点を選択して位置を変更（後述の頂点編集と同様）。
        - 穴または飛び地の輪郭上をクリックして新しい頂点を追加。

- **穴/飛び地の削除操作**:
    - 穴または飛び地のリングに属する頂点を選択してコンテキストメニューまたは削除キーで削除する。リングが3頂点未満になった場合、そのリング自体が削除される。
    - リングが削除される際の挙動は以下の通り：
        - **交互構造の保持**: リング階層は常に「領土リング → 穴リング → 領土リング → …」と交互になるよう維持する。付け替えや削除はこの不変条件を前提に行う。
        - **穴リング削除時の再編**: 対象が穴リングの場合、当該穴と直下の領土リングをまとめて削除する。削除される領土リングが保持していた穴リング子は、対象穴の親（必ず領土リング）へ付け替えて残す。例: 飛び地A → 穴A → 飛び地B → 穴B → 飛び地C で穴Aを削除すると、飛び地A → 穴B → 飛び地C となる。
        - **領土リング削除時の再編**: 対象が領土リングの場合、当該領土リングとその直下の穴リングを削除し、各穴が保持していた領土リング子を対象領土の親に付け替えて昇格させる。親が存在しない場合は最上位リングとして残す。例: 飛び地A → 穴A → 飛び地B → 穴B → 飛び地C で飛び地Aを削除すると、飛び地B → 穴B → 飛び地C が残る。
        - **操作の一体性**: これらの削除と再編は単一の編集操作として扱われ、アンドゥ／リドゥで完全に復元できること。
    - **制約（編集時のリアルタイム検証）**:
        - 穴リングは、それを直接含む親リング（外周または別の穴の中の飛び地）の内部に完全に含まれる必要がある。
        - 穴リングの境界と、親リングの境界が交差してはならない。
        - 同じ親を持つ兄弟リング（穴同士、または飛び地同士）が交差してはならない。
        - 飛び地は、他の外周リング（同ポリゴンの他の飛び地や、同レイヤーの他のポリゴン）と重なってはならない（これは「2.2.3 編集時の時間的・空間的整合性維持プロセス」で処理）。

- **面情報の排他性**:
    - 同じレイヤー内にある面情報と面情報は、決して重ならない（オーバーラップしない）。
        - これはドメインの基本ルールである。
        - 境界の接触（共有頂点・共有辺）は重なりに含めない。分割後に境界が一致するケースは許容する。
    - 特定の時間における面編集時（頂点移動、形状変更など）、以下の制約をリアルタイムで適用：
        - 同じレイヤー内の他の面情報の内部に頂点をドラッグしようとすると、エッジによって阻まれる。
        - エッジに沿って「滑る」動作を実装（カーソルがモニター端に達したときの挙動と同様）。
        - 頂点が滑っている境界は視覚的にハイライト表示する（色の変更や太さの増加）。
        - エッジ間の衝突判定も実装し、尖った部分を「飛び越える」ような動きでも重なりが発生しないようにする。
    - 面情報追加時（頂点配置中）、他の面情報の内部には頂点を設置できない。
    - 線情報と点情報は面情報との干渉を受けない（重なることが可能）。
    - 線情報と点情報に排他性はない。
    - **変更によって過去または未来で発生した排他性違反**は、後述の「2.2.3 編集時の時間的・空間的整合性維持プロセス」プロセスによって処理される。

- **衝突処理の詳細アルゴリズム（特定時間内の編集時）**:
    - エッジ沿いの滑り処理:
        - 頂点がエッジに接触した時点で、エッジ上への垂直投影点を計算
        - ドラッグ方向とエッジの方向ベクトルの内積を使用して、エッジに沿った移動方向を決定
        - エッジに沿って移動する際、エッジの端（頂点）に到達した場合:
            1. 接続された次のエッジがあり、ドラッグ方向が次のエッジへ進む方向であれば、次のエッジへ自動的に遷移
            2. 袋小路（行き止まり）の場合、それ以上の移動を制限
    - 衝突判定アルゴリズム:
        - 前処理: 空間分割（四分木など）を用いて衝突判定対象を絞り込み
        - 多角形間の交差判定:
            1. 辺同士の交差チェック: すべての辺の組み合わせで交差判定（計算量削減のため空間分割を活用）
            2. 包含チェック: 一方の多角形が他方に完全に含まれるケースの検出
            3. 自己交差チェック: 編集後の多角形が自己交差していないことを確認
            4. リング間の整合性チェック: 頂点移動によりリング間の包含関係や交差関係が不正にならないことを確認（例: 穴が外周の外に出る、兄弟リングが交差するなど）。
        - リアルタイム衝突回避:
            1. 移動先位置で仮想的にポリゴンを再構築（穴・飛び地を含む）
            2. 他ポリゴンとの衝突判定を実行
            3. 衝突が検出された場合、最も近い有効位置（衝突しない位置）を計算し、そこに移動
            4. 有効な移動先がない場合は、移動を抑制

- **共有頂点システム**:
    - 複数の面情報や線情報が同じ頂点を持つことでその境界を連動させる。
    - 共有された頂点の特徴：
        - 視覚的に通常の頂点と区別（異なる色、サイズなど）。
        - ドラッグ中、このまま離すと共有頂点化される状態では頂点と関係する辺のスタイルが変化する。
        - ある頂点を画面上の距離で他の頂点に近づけることで共有頂点になる。
            - 近接判定は絶対座標ではなくスクリーン座標で行い、同じ頂点のペアでも共有化されるかどうかはズームレベルに応じて変化する。
            - デフォルトのスナップ距離は50px。
            - スナップ距離は設定画面から変更できる。
            - 正確に言うと、近接判定True→スナップ→スナップされる距離のまま確定→共有頂点化
            - 共有化確定時は、ドラッグしていない側の頂点座標を保持し、ドラッグ中の頂点はその座標へ揃う。
            - **共有頂点化の参加制限（形状コンポーネント単位）**:
                - 同一リング内の頂点同士を同じ共有頂点グループに含めてはならない。
                - 同一線形内の頂点同士を同じ共有頂点グループに含めてはならない。
                - 同一地物であっても別リングに属する頂点同士は、候補から一律除外しない。
            - **幾何妥当性による確定制限**: 共有頂点化または共有頂点移動の結果、以下のいずれかが発生する場合は操作を禁止する:
                - ゼロ長エッジまたは連続重複頂点
                - リングの自己交差
                - 穴・飛び地の階層関係の破綻（穴が外周を突き抜ける等）
                - 同一レイヤー内の面重複
                    - これらの整合性検証は独立では行わず、別途定義される整合性維持の仕組みを利用する。
            - スナップされただけの状態で離すと、共有頂点化されずに通常の頂点移動になる。
        - 共有頂点をドラッグすると、その頂点を共有するすべての形状が連動して変化。
            - 共有頂点移動時の排他性制約：
                - 共有頂点の移動により同一レイヤー内で共有ポリゴン同士が重なる状況は原則として発生しない（すべての共有ポリゴンが同時移動するため）
                - 共有頂点の移動により、リング間の整合性が崩れる状況（穴が外周と交差するなど）も発生しない。
                - 異なるレイヤーに属する共有ポリゴン間では重なりが発生する可能性があるが、これは許容される
                - 共有ポリゴン群と非共有ポリゴン間の重なりが発生する場合は、エッジ滑り処理が適用される
        - 共有頂点のコンテキストメニューには「共有解除」オプションがある。
            - 共有解除すると、同じ座標に複数の独立した頂点が存在する状態になる。
        - 移動前の距離が一定以下である二点は、ドラッグ開始直後の自動共有化を抑制する（誤操作防止）。
            - 共有解除直後の頂点間がすぐに共有化されてしまわないための仕様。
            - そのドラッグ中に一度スナップ距離より外へ離れた時点でこの抑制は解除され、再び近づければドロップ前でも共有化候補になる。

- **多層レイヤー構造**:
    - 情報は複数階層（レイヤー）で管理できる。
    - レイヤーの特性：
        - レイヤーは序列（階層）を持ち、この順序は固定（入れ替え不可）。
        - 各レイヤーには一意の識別子と名前がある。
        - すべてのオブジェクト（点、線、面）は一つのレイヤーのみに属する（複数レイヤーに属さない）。
        - レイヤーの表示/非表示を個別に切り替え可能。
        - 表示中のレイヤーは、現在フォーカスされていないレイヤーは薄く表示。
    - レイヤーの追加と削除：
        - レイヤーの追加はプロジェクト設定ダイアログから行う。誤操作を防ぐため、マップ操作中に直接アクセスできる場所には配置しない。
        - レイヤーの削除は、全時刻を通じてそのレイヤーに所属する地物が存在しない場合にのみ許可する。地物が存在するレイヤーの削除は禁止する。
        - 削除実行前に確認ダイアログを表示する。
    - レイヤー間の関係：
        - 「上位レイヤー」：対象レイヤーより上位の階層にあるレイヤー。
        - 「下位レイヤー」：対象レイヤーより下位の階層にあるレイヤー。
        - 「上位領域」：面情報が属する上位レイヤーの面情報。
        - 「下位領域」：面情報に属する下位レイヤーの面情報。
    - レイヤー間の制約：
        - 面情報は、同一時点において高々1つの上位レイヤー面情報にのみ属する。面情報は親を持たなくてもよいが、複数の親を同時に持つことはできない。
        - 下位領域を持つ面情報は、その形状を直接編集できない。
        - 面情報の形状は、下位領域の和として計算される（下位領域がある場合）。
        - 下位領域を持つ面情報は頂点情報（自身のリング）を保持せず、常に下位領域の和として描画される。
        - 下位領域をすべて喪失した場合、上位領域も自動的に消失する。
    - レイヤー表示の操作：
        - 離散的なレイヤースライダーで表示レイヤーを切り替え。
        - 下位レイヤー表示時、そこに面情報がない場合は上位レイヤーの形状を流用。
        - レイヤー設定パネルでの詳細な表示設定が可能。

- **分裂と割譲/合邦機能**:
    これらの操作は、特定の作中時間において、一つまたは複数の地物の歴史の錨を生成・変更・削除する複合的な編集操作として扱われる。操作完了時には「2.2.3 編集時の時間的・空間的整合性維持プロセス」がトリガーされ、結果として生じる空間的オーバーラップ等がユーザーの選択に基づき解決される。
    - **分裂機能**：
        - 面情報を複数の部分に分割する操作。
        - 分割方法：
            - **二分割**：
                - ナイフツールで面を横断する分断線を作成する。
                - 操作確定時、元の面情報は対象時間で終了（既存の錨更新を含む）し、2つの面情報が生成される。
                - 既存プロパティをどちらの面情報に引き継ぐかは、分割確定後の選択ダイアログで指定する。
                - 選択されなかった面情報は新規プロパティ入力（または既定値）を持つ。
                - 確定後は「2.2.3 編集時の時間的・空間的整合性維持プロセス」と競合解決ダイアログに接続する（詳細は別節で定義）。
            - **穴あけによる新地物生成**：ユーザーが面情報内部に新しい領域（穴の形状）を指定する。操作確定時、その領域が新しい独立した面情報として生成され、元の面情報には対応する穴が形成される。それぞれの地物の新しい状態が歴史の錨としてその作中時間から記録される。
                - これは穴追加機能とは異なる。穴追加機能であれば同じに地物にリングが追加され、穴あけでは地物は2つできる。
        - 分割条件：
            - 下位領域がない面情報（自身のリングで形状が定義されている面情報）のみ分割可能。
            - 分割後の各面情報は、最低3頂点以上で有効なポリゴン形状を形成する必要がある。
    - **割譲と合邦機能（所属変更）**:
        - 面情報のドメイン階層における所属関係（親領域識別子および対応する親の下位領域識別子一覧）を、特定の作中時間から変更する操作。
        - 操作方法：
            - 面情報を選択し、コンテキストメニュー等から「所属変更」を選択。
            - 新しい上位領域（親となる面情報）を選択するダイアログを表示。または「親なし（最上位領域へ）」も選択可能。
        - この操作は、対象面情報の特定の作中時間以降の全ての歴史の錨において親領域識別子を更新し、関連する旧親および新親の下位領域識別子一覧も同様に更新する形で新しい歴史の錨を定義または既存の錨を更新する。
        - **割譲**: 下位領域が、既存ではない新しい上位領域（ユーザーが操作中に新規作成を指定可能）に属するように変更することで、上位領域の事実上の分裂（一部が新しい上位領域として独立）を表現する。
        - **合邦**: ある面情報（旧親A）の全ての下位領域が、別の既存の上位領域（新親B）に一括して所属変更することで、親Aが親Bに吸収されたかのような合邦を表現する。この際、旧親Aのその作中時間以降の存在や形状は、下位領域を失った結果として再計算された歴史の錨として更新される。
        - **飛び地化の許容**: 下位領域は、地理的に陸続きではない上位領域にも所属することが可能。この場合、上位領域の形状は、その歴史の錨において、地理的に分離した全ての下位領域を包含するように（複数の独立したリングを持つ形で）表現される。
        - 制約：
            - 面情報が自身に下位領域を持たない場合（リーフノードである場合）にのみ、直接の所属変更が可能。下位領域を持つ面情報の所属を変更する場合は、まずその全ての下位領域の所属を変更する必要がある。
            - レイヤー間の移動は自由（レベル5からレベル1への昇格なども可能）。

- **合体（結合）機能**:
    - **機能**: 同じレイヤーに属する複数の面情報を、特定の時間において1つの新しい面情報に統合する操作。
    - **操作方法**:
        - 編集モードで「結合」ツールを選択。
        - 結合したい複数の面情報を選択（Shiftキー+クリックなど）。
        - コンテキストメニューまたはツールバーのボタンから「結合」を実行。
    - **結合処理**:
        - 選択された面情報が、操作対象の時間において存在し、かつ地理的に隣接しているか（共有境界線/頂点を持つか）をシステムが評価する。隣接していない場合は、結果として飛び地を持つ単一の面情報として統合される。
        - 結合後の新しい単一の面情報のプロパティ（名前、属性など）をユーザーが入力するダイアログを表示。
        - 元となった複数の面情報は、その時間において存在を終了する歴史の錨が打たれる。
        - 新しい結合済み面情報は、その時間から始まる新しい歴史の錨として記録される。その形状は、元となった面情報の形状の論理和（ブーリアン演算の結合）として計算される。既存の穴や飛び地などの構造は適切に維持・統合される。
        - 違う上位領域に属する面情報同士での結合である場合、結合後の面情報がどれに属するかを決定するダイアログを表示。ユーザーは以下のいずれかを選択：
            - 結合する面情報のいずれか1つの属する上位領域に属する。
                - リスト状のUIを提供し、そのうち一つを選ぶ
            - 新しい上位領域を作成する。
            - どの上位領域にも属さない。
        - 上位領域の決定後、結合後の面情報のプロパティを決定するダイアログを表示。ユーザーは以下のいずれかを選択：
            - 結合する面情報のいずれか1つのプロパティを引き継ぐ。
                - リスト状のUIを提供し、そのうち一つを選ぶ
            - 新しいプロパティを入力する。
        - この操作も完了時に「2.2.3 編集時の時間的・空間的整合性維持プロセス」を経る（主に他の地物との空間的オーバーラップの可能性について）。
    - **制約**:
        - 同じレイヤーに属する面情報のみ結合可能。
        - 結合対象の面情報が、操作対象の時間において下位領域を持たないこと。

### 2.2 時間管理機能

#### 2.2.1 「歴史の錨」による状態定義

地理的オブジェクト（以下、地物）の経時的状態は、歴史の錨（以下、錨）によって定義される。

- **錨の定義**: 各歴史の錨は、特定の時刻における地物の完全な状態スナップショット（形状、属性、親子関係、存在状態など）を記録する。具体的には、地物ごとに時系列で保持される状態記録の各要素が、一つの歴史の錨として機能する。
- **地物のライフサイクルと錨**:
    - **生成**: 地物の最初の歴史の錨が定義された開始時刻が、その地物の存在開始時点となる。開始時刻以前にその地物は存在しないものとして扱われる。
    - **状態変更**: 地物の形状、属性、親子関係などが変化する各時間に、新しい歴史の錨が定義される。
    - **消滅**: 地物の最後の歴史の錨に終了時刻が設定される場合、その状態が地物の最終状態となり、終了時刻以降は存在しないものとして扱われる。

#### 2.2.2 歴史の錨の時間的継続と優先順位

- **状態の継続**: ある歴史の錨Aで定義された地物の状態は、その開始時刻から、同一地物に対する次に新しい歴史の錨Bの開始時刻の直前まで継続するものとする。この期間中、地物の状態は錨Aで定義されたものと同一である。
- **過去の歴史の錨の編集による未来への影響**: ユーザーが過去の時刻に設定された歴史の錨を編集した場合、その編集内容は当該時刻以降のその地物の状態の基礎となる。この影響は、同一地物に対する次の未来の歴史の錨が存在するならば、その錨の開始時刻の直前まで継続する。
- **未来の歴史の錨の保護**: 未来側の歴史の錨で定義された状態（形状、属性など）は、過去側の錨の編集によって**自動的に内容が書き換えられることはない**。未来側の錨は、その開始時刻におけるユーザーの明示的な状態定義として尊重される。過去の編集による状態の伝播は、未来側の錨の開始時刻で遮断される。
- **同一地物の錨列の制約**: 同一地物に属する歴史の錨は、開始時刻の昇順で管理し、時間的な重複を禁止する。
- **時間的飛び地（非存在期間）**: 錨同士が連続していない期間（ある錨の終了時刻から次の錨の開始時刻までの空白期間）を許可する。空白期間は、その地物が存在しない期間として扱う。

##### 2.2.2.1 過去編集時の錨更新ルール

- **編集時刻の定義**: 編集の基準時刻は、タイムラインで現在選択されている時刻とする。
- **同時刻の錨が存在する場合**: 編集時刻と一致する開始時刻を持つ歴史の錨が存在する場合、その錨を更新する。
- **同時刻の錨が存在しない場合（時間点での分割）**: 編集時刻と一致する錨が存在しない場合、編集時刻を開始時刻とする新しい錨を追加する。新しい錨の初期値は、編集時刻で有効な錨の状態を引き継ぎ、ユーザーの編集内容を反映する。
- **有効期間の扱い**: 新しい錨は編集時刻を開始時間とし、次の未来の錨が存在する場合はその開始時刻の直前まで有効とする。未来の錨の内容は一切変更しない。
- **錨境界の編集（開始時刻・終了時刻）**:
    - ユーザーは、選択中の歴史の錨について、開始時刻および終了時刻を変更できる。
    - 境界変更により同一地物内で時間重複が発生する場合、システムは対象地物の錨列を自動で再編し、重複期間を解消する（必要に応じて既存錨の分割・短縮・削除を行う）。
    - 境界変更後の錨列は、開始時刻昇順・重複なし・各錨の有効期間が正の長さ、の条件を満たさなければならない。
    - いずれの再編でも、対象地物の錨列が0件になる結果は許可しない。
- **終了時刻の制約**:
    - 終了時刻を指定する場合は、当該錨の開始時刻より後でなければならない。
    - 次の未来の錨が存在する場合、終了時刻はその開始時刻と同時刻まで、またはそれより前に設定できる。同時刻より前に設定した場合、その間は非存在期間（時間的飛び地）として扱う。
    - 次の未来の錨が存在する場合、終了時刻を次の未来の錨の終了時刻よりも後にすることはできない。
- **編集対象の粒度**:
    - 属性のみを編集する場合でも、システム内部では当該錨の完全状態（形状・属性・所属・親子関係を含む）を一体として扱う。
    - 形状を編集する場合は、同一時刻の錨の形状を置換し、必要に応じて同時刻の属性・所属・親子関係と整合するよう再構成する。
    - これらの編集は、保存確定時に単一の編集案として扱い、部分的な確定を許可しない。

##### 2.2.2.2 錨境界編集の例

- **建国時期の前倒し（開始時刻変更）**:
    - 例として、ある地物に「N年開始〜M年終了の錨」と「M年開始〜O年終了の錨」が存在する状態を考える。
    - 後から「実はN-300年から存在していた」ことが判明し、ユーザーが前者の錨の開始時刻をN-300年へ変更する操作を行ったとする。
    - この境界編集の結果は、通常は「N-300年開始〜M年終了の錨」として反映される。
    - ただし整合性プロセスで別地物と領域の競合が検出され、例えばN-100年を境界に解決が必要になった場合は、N-300年〜N-100年の錨と、N-100年〜M年の錨へ自動分割されうる。
- **探検・滅亡時期の変更（開始/終了境界の移動）**:
    - M年（領域変化の開始時刻）やO年（滅亡時刻）の変更も同様に、錨境界の変更として扱う。
    - 境界変更の結果は、同一地物内の錨列を再編した上で、時間整合性と空間整合性を満たす形で確定される。
- **復活（時間的飛び地）**:
    - ある地物が一度消滅し、復活する場合、消滅期間を非存在期間（錨の空白期間）として表現できる。
    - 錨の表示は、二種類の表示方法を切り替えられるようにする。
        - 非存在期間を空白として表示し、上端をタイムラインの開始時間、下端を終了時間とする「時間バー表示」
            - 錨は存在している時間に当たる部分にだけ表示され、そこをクリックすると各錨が選択される。
            - マウスホイールでズームが可能。100％表示以上の時、スクロールバーが表示される。
            - 100以下にズームアウトすることはできない。
            - 時間バーでは、固定の開始時間ラベル・終了時間ラベルは表示しない。
            - 代わりに、現在表示中の領域の上端時刻と下端時刻を表示し、ズームおよびスクロールに応じて動的に更新する。
            - 錨名称ラベルは、表示中領域に対する画面占有率に基づいて表示可否を判定し、はみ出しや重なりが生じる場合はラベル表示を抑制する。
        - 単純に錨を列挙した「リスト表示」

#### 2.2.3 編集時の時間的・空間的整合性維持プロセス

ユーザーが歴史の錨を追加または既存の歴史の錨を編集し、その操作を**確定しようとした際**には、システムは時間的・空間的なデータの整合性を維持するため、以下の検証と解決のプロセスを実行する。
このプロセスは、本節内の「競合の検出と解決」セクションで詳細に定義される。

- **保存前フェーズの固定順序**:
    1. 編集案の生成（編集対象錨の確定、境界変更の反映、必要な錨分割の適用）
    2. 錨の時間整合性検証（同時刻重複・期間逆転・未来錨境界超過の禁止）
    3. 空間競合検出（同一レイヤー面情報の重なり検出）
    4. 競合解決ダイアログによる方針指定
    5. 解決適用と一括確定（競合境界で必要になった錨分割を含む。単一のアンドゥ単位）

- **競合の検出**: 操作の結果として生じる可能性のある以下の競合を検出する。
    1. **空間的オーバーラップ**: 同一レイヤーに属する複数のポリゴン地物が、ある作中時間範囲において空間的に重なり合う状態。
- **競合解決ダイアログによるユーザー選択**: 空間的オーバーラップが検出された場合、専用の解決ダイアログが起動される。ユーザーは提示された各競合箇所について、ミニマップでのプレビューを参考に解決方針（例: どちらの地物の形状を優先し、もう一方をブーリアン演算で調整するか）を選択する。
- **解決の適用と確定**: 全ての競合についてユーザーが解決方針を指定した後、システムはその指示に従って関連する全ての歴史の錨を調整し、編集操作全体を確定する。またはキャンセルされる。
- **競合解決ダイアログ**:
    - 上記の競合が一つでも検出された場合、編集操作の確定は一時停止され、専用のモーダルダイアログが表示される。
    - ダイアログには、検出された全ての競合箇所（作中時間または時間範囲、関連する地物名、競合の種類）がリスト形式で表示される。
    - ユーザーはリストから競合箇所を一つずつ選択する。
        - 選択すると、ダイアログ内にその競合状況を視覚的に示すミニマップ（該当する作中時間、該当箇所がちょうど映るようにズームアップ）と、関連情報が表示される。
    - 各競合箇所について、ユーザーは以下の解決方針を選択する（競合の種類に応じて選択肢は変わる）。
        - オプション1（地物Xを優先）: 地物Xの形状を維持し、競合相手の地物Yの形状を、地物Xと重ならないようにブーリアン演算（差分 Y - X）によって自動調整する。
        - オプション2（地物Yを優先）: 地物Yの形状を維持し、競合相手の地物Xの形状を、地物Yと重ならないようにブーリアン演算（差分 X - Y）によって自動調整する。
    - ユーザーがリストアップされた全ての競合について解決方針を指定し終えるまで、元の編集操作は確定されない。
    - 全ての競合解決方針が指定された後、「解決を適用して編集を確定」ボタンを押すと、システムは指定された全ての方針に従って関連する歴史の錨のプロパティ（形状、期間、属性、イベント定義メタデータなど）を更新し、一連の編集をデータに反映する。
        - この一連の解決と編集の適用は、単一のアンドゥ・リドゥ操作の対象となる。

#### 2.2.4 タイムラインインターフェース

- **時間軸スライダー**:
    - ユーザーはスライダーを操作することで、表示対象の時間を直感的に変更できる。スライダーの移動に応じて、メインマップの表示は即時に該当時間の状態に更新される。
    - スライダーには、設定された時間範囲（最小年、最大年）と、現在の時間が数値で表示される。
- **数値入力**: ユーザーは特定の時間（年、オプションで月・日）を数値で直接入力し、その時点にジャンプできる。
- **時間単位と紀元前/紀元後対応**: システムは年を基本単位とし、必要に応じて月・日レベルの精度を扱う。負の年数を紀元前として解釈・表示する。
- **時間範囲の設定**: アプリケーション全体、またはプロジェクトごとに、タイムラインスライダーがカバーする時間の最小値と最大値を設定可能（デフォルトは0年から10000年）。
- **時間軸の目盛りとマーカー**: スライダー下部には、主要な年代を示す目盛りとラベルが表示される。ユーザーは、重要な歴史的イベントが発生した時間を「イベントマーカー」として時間軸上に表示・設定できる。

##### 2.2.4.1 時間粒度の入力手段

- 年・月・日をそれぞれ独立した数値入力欄として提示する。年は整数で入力し、月・日は 1 桁の場合でもゼロ埋めせず表示する。
- 月・日の各入力欄には「未指定」のトグルを設け、未指定が選択された場合は当該粒度の情報を保持しない状態として扱う。保存済みの時間情報が粒度未指定である場合も同じ表示を採用する。
- 入力値の妥当性は暦設定に基づいて検証し、無効な値は即時にエラー表示して確定させない。初期実装ではグレゴリオ暦を前提とし、独自歴への拡張は将来実装（第7章参照）とする。
- 数値入力はフォーカスアウトまたはエンターキーで確定し、確定時に現在時刻が更新され、すべてのタイムライン関連表示へ即時反映される。

##### 2.2.4.2 ステップ操作の単位

- 前進・後退ボタンの近くに単位選択を配置し、年・月・日のいずれかを選択できるようにする。初期状態は年とする。
- 月または日を選択した状態で月・日が未指定のときは、暦設定に基づく既定値（1 月 1 日）を自動補完し、表示と内部状態を整合させてから操作を継続する。
- 年単位を選択した場合は現在の年数に対して整数値で加減算し、月または日を選択した場合は暦設定に基づく日数換算で加減算する。
- ステップ値は常に 1 単位とし、ボタンの長押しで連続操作を行える。

##### 2.2.4.3 再生制御

- 再生は 100 ミリ秒間隔のタイマーで駆動し、各ティックで内部の時間更新処理を日数単位で実行する。
- 再生速度セレクタは「0.1倍 / 0.5倍 / 1倍 / 2倍 / 5倍 / 10倍」を提供し、1倍を1年/秒と定義する。暦設定の年間日数に基づき、選択された倍率をティックごとの日数増分へ換算する。
- 月・日が未指定の状態で再生を開始する場合は、2.2.4.2 と同じ補完処理を行ったうえで再生を続行する。再生中に日単位の加算結果として月・日が確定した場合は表示へ即時反映する。
- 時間範囲の最小値または最大値に到達した際は自動停止し、停止時には現在時刻と再生状態の変化を関係コンポーネントへ通知する。

##### 2.2.4.4 他機能との連携

- プロパティ編集パネルの存在期間入力はタイムラインと同じ入力コンポーネントを共有し、年必須・月日任意という粒度を統一する。
- 時間変更に関する通知は常に年・月・日を個別の値として配信し、小数年などの中間表現を使用しない。
- タイムラインに関連するすべての時間操作は、暦設定と整合する共通の時間計算ロジックを経由して実行する。

#### 2.2.5 時間依存データの視覚化と正規化

- **状態の視覚化**: タイムラインスライダーの操作（または他の時間変更操作）に応じて、メインマップに表示される全ての地物の状態（形状、属性、存在の有無）は、選択された時間における有効な歴史の錨に基づいて即時に更新される。
- **時間表現の正規化**:
    - システム内部では、時間は一貫した基準（例: 通日、または詳細な日付時刻オブジェクト）で処理される。
    - ユーザーインターフェース上では、時間は年のみ、年月、年月日の各レベルの精度で表示・編集が可能であり、入力された情報は内部で正規化される。
    - 全ての時間比較、期間計算、状態の継続判定などは、この正規化された時間表現に基づいて行われる。

#### 2.2.6 履歴データ形式の方針

- 歴史の保存形式は、錨単位で完全状態を保持する方式とする（詳細は第4章）。

## 2.3 オブジェクト編集機能

### 2.3.1 アンドゥ・リドゥ

- **対象範囲**: すべての編集操作（歴史の錨の追加・変更・削除、およびそれに伴う競合解決プロセスの結果全体を含む）に対応する。
- **操作粒度**:
    - 単一のユーザーアクション（例: 頂点ドラッグ完了、プロパティ保存、競合解決ダイアログの確定）をアンドゥの単位とする。
    - 競合解決を含む複合操作（分割＋競合解決＋確定）であっても、一連の操作全体が単一のアンドゥ単位として扱われる。
- **履歴管理**:
    - アンドゥスタックとリドゥスタックの二重スタック構造で履歴を管理する。
    - 履歴の深さは最大100操作まで保持する（設定により変更可能）。
    - アンドゥスタックが上限に達した場合、最も古い操作から順に破棄する。
    - 新しい編集操作が実行された場合、リドゥスタックは全て破棄される。
- **UI操作**:
    - **アンドゥ**: Ctrl+Z（Windows）または Command+Z（Mac）。
    - **リドゥ**: Ctrl+Y（Windows）または Command+Shift+Z（Mac）。
    - 編集メニューからも「元に戻す」「やり直す」として操作可能。
    - アンドゥ/リドゥ可能な操作がない場合、対応するメニュー項目およびショートカットは無効化される。

### 2.3.2 追加モード

- **モード切替**:
    - ツールバーのトグルボタンで「追加モード」をON/OFFする。
    - 追加モード有効時は、関連する追加ツール群がツールバーに表示される。
    - 追加モードと編集モードは排他的であり、一方を有効にすると他方は自動的に無効になる。

- **点情報追加ツール**:
    1. ツールバーから「点情報追加」ツールを選択する。
    2. 地図上の任意の位置をクリックして点を配置する。
    3. 配置と同時にプロパティ入力フォーム（名前、説明、年代等）が表示される。
    4. プロパティ入力を完了し保存すると、現在のタイムライン時刻を開始時刻とする歴史の錨として記録される。
    5. 記録後、「編集時の時間的・空間的整合性維持プロセス」がトリガーされる。

- **線情報追加ツール**:
    1. ツールバーから「線情報追加」ツールを選択する。
    2. 地図上を連続クリックして頂点を追加する。
    3. 2点以上で線として成立し、確定ボタンが有効化される。
    4. 確定ボタンの押下またはダブルクリックで形状を確定する。
    5. 確定後にプロパティ入力フォーム（名前、説明、年代等）が表示される。
    6. プロパティ入力を完了し保存すると、現在のタイムライン時刻を開始時刻とする歴史の錨として記録される。
    7. 記録後、「編集時の時間的・空間的整合性維持プロセス」がトリガーされる。

- **面情報追加ツール**:
    1. ツールバーから「面情報追加」ツールを選択する。
    2. 地図上を連続クリックして頂点を追加する。
    3. 3点以上で面として成立し、確定ボタンが有効化される。
    4. 確定ボタンの押下またはダブルクリックで形状を確定する。
    5. 確定後にプロパティ入力フォーム（名前、説明、年代等）が表示される。
    6. プロパティ入力を完了し保存すると、現在のタイムライン時刻を開始時刻とする歴史の錨として記録される。
    7. 記録後、「編集時の時間的・空間的整合性維持プロセス」がトリガーされる（特に同一レイヤーの他面情報との空間的オーバーラップを検出）。
    8. 追加中の頂点配置時には、リアルタイムで同一レイヤー同一時間の他の面情報の内部には頂点を設置できない（面情報の排他性制約）。

- **追加時の共通機能**:
    - **仮表示**: 追加中のオブジェクトは、確定済みオブジェクトとは異なるスタイル（点線、半透明など）で仮表示される。
    - **クロージング線**: 面情報追加時、頂点が2つ以上配置された状態では、最後に配置した頂点から最初の頂点へ向かう閉合線を半透明で表示する。これにより確定後のポリゴン形状を視覚的に把握しやすくする。
    - **取り消し**: 最後に追加した頂点を1つずつ削除できる（Ctrl+Z または専用ボタン）。
    - **キャンセル**: 追加作業全体をキャンセルし、仮表示を破棄して追加開始前の状態に戻る（Escape キーまたは専用ボタン）。
    - **グリッドスナップ**: グリッド線へのスナップ機能をオプションで利用可能。有効時は頂点がグリッド交点に吸着する。

### 2.3.3 編集モード

- **モード切替**:
    - ツールバーのボタンで「編集モード」に切り替える。
    - 編集モード有効時は、関連する編集ツール群がツールバーに表示される。

#### 2.3.3.1 選択機能

- **アクティブ地物**:
    - ユーザーが地物本体を直接操作して明示的に選択した地物を指す。
    - 常に0件または1件のみ保持される。
    - アクティブ地物が設定されると、当該地物の全頂点が半透明マーカーで表示される。

- **頂点選択セット**:
    - 現在選択されている頂点IDの集合。
    - Shiftキーを押しながらクリックすることで頂点の追加・除去が可能。
    - 単一クリック（Shiftキーなし）はセット全体を置き換える。
    - 範囲選択（ドラッグによる矩形選択）で複数頂点を一括選択可能。
    - 複数頂点が選択されている状態で、その選択済み頂点のいずれかをドラッグした場合、頂点選択セット全体を同一ベクトルで同時移動する。
    - この同時移動は単一の編集操作として扱い、アンドゥ／リドゥでまとめて復元できること。
    - 同時移動中に共有頂点が含まれる場合、その共有関係にある頂点も同一ベクトルで連動して移動する。
    - 複数頂点の同時移動中は、新たな共有頂点化の確定は行わない。

- **頂点選択コンテキスト**:
    - 頂点選択セットに含まれる頂点の所有者（地物）を解析する。
    - 単一の地物だけが全頂点を所有している場合に限り、その地物IDを保持する。
    - 複数の地物にまたがる場合は「未確定状態」とする。
    - プロパティパネルなどはアクティブ地物が無い場合に限り、このコンテキストを参照する。

- **クリック動作**:
    - **地物（頂点以外）をクリック**: その地物をアクティブ地物として設定し、頂点選択セットと頂点選択コンテキストをクリアする。
    - **頂点をクリック**: 頂点選択セットを更新する。Shiftキーを押さない単一クリックはセットを置き換え、Shiftキーを押しながらクリックすると選択の加除を行う。この操作を行った瞬間にアクティブ地物は解除される。
    - **何もない場所をクリック**: アクティブ地物と頂点選択セットをすべて解除する。

- **範囲選択**:
    - マウスドラッグで矩形選択範囲を作成し、範囲内の頂点や地物の頂点を複数選択する。
    - Shiftキー併用で既存の選択に追加する。

- **選択解除**:
    - Escapeキーでアクティブ地物、頂点選択セット、頂点選択コンテキストをまとめて解除する。

- **視覚的フィードバック**:
    - **アクティブ地物**: 太いシアン（水色）のアウトラインとシアンの塗りで強調表示する。
    - **頂点選択コンテキスト地物**: 頂点選択コンテキストが存在し、かつアクティブ地物と異なる場合は、その地物を細い点線アウトラインで示す。
    - **選択済み頂点**: 頂点選択セットに含まれる頂点は、強調表示され、ドラッグ中の頂点とは別スタイルで区別される。

- **選択地物の頂点表示**:
    - アクティブ地物がある場合、その形状を構成するすべての頂点を半透明マーカーで表示する。
    - 選択済み頂点およびドラッグ対象の頂点はより目立つスタイルを維持する。

- **プロパティ表示**:
    - **アクティブ地物がある場合**: サイドバーのプロパティタブにその地物のプロパティを表示する。
    - **アクティブ地物が無く、頂点選択コンテキストがひとつに定まっている場合**: その地物のプロパティを表示する。
    - **歴史の錨一覧**: プロパティタブ内には歴史の錨一覧を表示し、ユーザーは編集対象の歴史の錨を明示的に選択できる。
        - 歴史の錨一覧で歴史の錨を選択した場合、タイムライン表示と現在時刻を当該歴史の錨の開始時刻へ即時同期する。
        - 同期後のプロパティ入力欄は、その開始時刻に対応する歴史の錨の内容を表示する。
        - 同期後の保存操作は、選択された歴史の錨の開始時刻を編集時刻として実行する。
    - **複数地物が検出された場合**: 頂点選択セットに含まれる頂点の所有者が複数存在する（頂点選択コンテキストが未確定で、所有地物数が2件以上の）場合は、「複数の地物が選択されています。プロパティ編集は1件ずつ行ってください。」と表示する。検出した地物名（最大5件、残数は件数で表示）とIDを列挙して単一選択への切り替えを促す。プロパティ編集フォームは生成しない。
    - **所有者不明の場合**: 頂点選択セットは存在するが可視レイヤー上の所有者が特定できない場合は、「選択された頂点の所有者を特定できません。可視レイヤーの地物を直接選択してください。」と表示する。フォームは生成しない。
    - **空選択の場合**: 上記いずれにも該当しない場合（選択が完全に空の場合）、プロパティタブには「地物または頂点が選択されていません」と表示する。

- **「地物を削除」ボタン**:
    - プロパティタブに常設する。
    - アクティブ地物または頂点選択コンテキストの地物が表示されている場合のみ有効化する。
    - 押下時は確認ダイアログで地物名と削除対象を提示し、ユーザーが確定した場合に限り削除処理ユースケース（履歴記録を伴う）を経由して地物全体を削除する。
    - キャンセルした場合は何も変更せず、ダイアログを閉じる。

- **地物ラベル表示**:
    - ラベル描画は、その地物に名前が定義されており、かつラベル表示設定が有効であることを前提とする。
    - 描画有無と配置の条件は地物種別ごとに以下の通り定義する。

    - **点情報ラベル**:
        - プロパティの「点情報ラベル表示最小ズーム倍率」が定義されている場合、その値以上にズームインしていなければラベルを描画しない。
        - 未定義の場合はプロジェクト設定に定められた最小ズーム倍率を使用する。
        - ラベルは点のスクリーン座標を基準に、シンボル半径に2px（ズームに合わせてスケーリング）を加算した距離だけ上方向へオフセットして配置する。
        - フォントサイズは、地物スタイルで指定された基本サイズを現在ズーム倍率で割った値を、5px〜16px の範囲に収めて決定する。

    - **線情報ラベル**:
        - 線をスクリーン座標へ投影した折れ線の可視長を計測し、その値に基づいて表示/非表示を決定する。
        - プロパティの「線情報ラベル表示最小表示長」が定義されている場合、表示長がその値未満の場合はラベルを描画しない。
        - 未定義の場合はプロジェクト設定に定められた表示長を使用する。
        - ラベルのアンカーは折れ線長に基づく中点とし、線上方向から読みやすいよう画面座標系の上方向へ「5px / 現在ズーム倍率」だけオフセットして描画する。
        - 可視長がしきい値を超える限り、1本の線地物につき1枚のラベルを描画する。

    - **面情報ラベル**:
        - 地物の各外周および飛び地のリングごとにラベルを描画する。ユーザーが外周リングのみを削除した場合でも、残存する飛び地に対応するラベルが表示される。
        - **画面占有率の計算**: 各リングでは、外周リング面積から直下の穴リング面積を差し引いた「有効面積」を算出する。これを「表示領域の幅 / 現在ズーム倍率」と「表示領域の高さ / 現在ズーム倍率」の積で割り、「画面占有率」として扱う。
        - 画面占有率が0.0005（表示領域の0.05%）未満のリングはラベルを描画しない。
        - 閾値はプロジェクト設定で調整可能とし、未指定の場合は0.0005を既定値とする。
        - ラベル表示可否の判定は、初期描画・通常再描画・ズーム確定後再描画で同一の計算を適用する。
        - ラベル配置は親リングから子リングの順に優先度を付け、同一地物内で重ならないよう外接矩形の中心を基準に必要なオフセット調整を行う。穴の内部に飛び地が存在する場合でも視認性が確保される位置に配置する。
        - ラベルが重なる恐れがある場合は、半径方向へのシフトや描画順で視認性を確保し、最終的にユーザーが読めるようにする。

- **頂点/エッジハンドルの表示**:
    - 選択された地物の形状を構成する各頂点は**頂点ハンドル**として、各辺（エッジ）の中点は**エッジハンドル**として、地図上に視覚的に表示される。
    - **頂点ハンドル**: 頂点の移動や削除の操作対象となる。選択状態や共有状態によってスタイルが変化する。
    - **エッジハンドル**: 新しい頂点を追加するための操作対象となる。

#### 2.3.3.2 編集ツール

- **地物移動ツール（平行移動）**:
    - アクティブ地物をドラッグして平行移動する。
    - 移動量は当該地物に紐づく全頂点へ同一ベクトルで適用する。
    - 共有頂点も移動対象に含め、共有先の地物にも位置変更が反映される。
    - 操作確定時に歴史の錨を更新し、「編集時の時間的・空間的整合性維持プロセス」をトリガーする。

- **複数頂点同時移動**:
    - 頂点選択セットに複数頂点が含まれる場合、選択済み頂点をドラッグすると、その集合全体を同一ベクトルで平行移動する。
    - 移動量はドラッグを開始した頂点の移動量を基準とし、他の選択済み頂点へ同一量を適用する。
    - 同時移動の確定時も、単一頂点移動と同様に「編集時の時間的・空間的整合性維持プロセス」を適用する。
    - 同時移動により共有頂点グループが影響を受ける場合、当該共有頂点グループの代表座標と参加頂点の座標を一貫して更新する。

- **点情報編集ツール**:
    - **属性編集**: 点のプロパティ（名前、説明、年代等）を変更する。完了時に歴史の錨を更新し、「編集時の時間的・空間的整合性維持プロセス」をトリガーする。
    - **移動**: 点をドラッグして位置を変更する。完了時に歴史の錨を更新し、「編集時の時間的・空間的整合性維持プロセス」をトリガーする。
    - **削除**: 点を選択してDeleteキーまたはコンテキストメニューから削除する。完了時に歴史の錨を削除し、「編集時の時間的・空間的整合性維持プロセス」をトリガーする（関連イベントへの影響）。

- **線情報編集ツール**:
    - **属性編集**: 線のプロパティ（名前、説明、年代等）を変更する。完了時に歴史の錨を更新する。
    - **頂点編集**（特定時間内での操作）:
        - **頂点表示**: 線上の各頂点がハンドルで表示される。
        - **頂点選択**: クリックで特定の頂点を選択する。
        - **頂点移動**: 選択した頂点をドラッグして移動する。
        - **頂点追加（エッジハンドル操作）**:
            - 選択された地物の各辺（エッジ）の中点には、エッジハンドルが視覚的に表示される。
            - ユーザーがこのエッジハンドルをドラッグすると、そのエッジ上に新しい頂点が追加される。
            - 具体的なフロー:
                1. ドラッグを開始した瞬間に、元のエッジが2つのセグメントに分割され、その間に新しい頂点が挿入される。
                2. ドラッグ中は、この新しい頂点がマウスカーソルに追従して移動する。
                3. マウスボタンを離した時点で頂点の位置が確定し、地物の形状が更新されたことが歴史の錨として記録される。
        - **頂点削除**: 選択された頂点をDeleteキーで削除する。2点未満になった場合、線自体が削除される。
    - 上記の頂点編集操作完了後、その時間における新しい形状で歴史の錨が更新され、「編集時の時間的・空間的整合性維持プロセス」がトリガーされる。

- **面情報編集ツール**:
    - **属性編集**: 面のプロパティ（名前、説明、年代等）を変更する。完了時に歴史の錨を更新する。
    - **頂点編集**（特定時間内での操作）:
        - **頂点表示**: 面の各頂点がハンドルで表示される。
        - **頂点選択**: クリックで特定の頂点を選択する。
        - **頂点移動**: 選択した頂点をドラッグして移動する。
            - 同一レイヤーの他面情報との衝突判定をリアルタイムで実施（エッジ滑りなど）。
            - 穴の頂点を移動している時、外周と交差しないように制約する。
            - 飛び地の頂点についても同様の制約を実施する。
        - **頂点追加（エッジハンドル操作）**:
            - 選択された地物の各辺（エッジ）の中点には、エッジハンドルが視覚的に表示される。
            - ユーザーがこのエッジハンドルをドラッグすると、そのエッジ上に新しい頂点が追加される。
            - 具体的なフロー:
                1. ドラッグを開始した瞬間に、元のエッジが2つのセグメントに分割され、その間に新しい頂点が挿入される。
                2. ドラッグ中は、この新しい頂点がマウスカーソルに追従して移動する。この移動中も、他の頂点と同様に、同一レイヤーの他ポリゴンとの衝突判定（エッジ滑り）や、他の頂点とのスナップ・共有化の対象となる。
                3. マウスボタンを離した時点で頂点の位置が確定し、地物の形状が更新されたことが歴史の錨として記録される。
        - **頂点削除**: 選択された頂点をDeleteキーで削除する。3点未満になった場合、面自体が削除される（または関連するリングが削除される）。
    - 上記の頂点編集操作完了後、その時間における新しい形状で歴史の錨が更新され、「編集時の時間的・空間的整合性維持プロセス」がトリガーされる。

- **穴/飛び地追加ツール**:
    - 2.1節（穴あき領域・飛び地の編集）で詳細記述済み。確定時に「編集時の時間的・空間的整合性維持プロセス」をトリガーする。

- **面情報結合ツール**:
    - 2.1節（合体・結合機能）で詳細記述済み。確定時に「編集時の時間的・空間的整合性維持プロセス」をトリガーする。

- **ナイフツール（二分割）**:
    - **概要**: 面情報を分断線によって2つの面情報に分割する。
    - **前提操作**: 事前に対象の面情報を選択する。選択中は地図全体を薄いオーバーレイで覆い、対象面だけを強調表示する。
    - **分断線の入力**:
        - 線情報追加と同じ操作感で分断線を入力する。
        - 入力開始と同時に確定/キャンセルの操作UIを表示する。
        - 確定ボタンは有効条件（分割が成立する形状）を満たすまで非活性とする。
        - キャンセル時は確認ダイアログを挟む。
        - **制約なし**: 分断線の点配置には制約を設けない。面内・面外・穴内のいずれにも点を配置できる。
        - 確定時に分割可能かどうかを検証し、成立しない場合はエラーメッセージを表示する。
    - **穴リング横切り時の処理**:
        - 分断線が穴リングを横切る場合、穴リングは2分割されて各新ポリゴンの外周リングへ合流する。
        - このとき穴は外周へ開放されるため、穴内部の飛び地は親リングを一段上げた飛び地となる。
        - 分断線が特定のリング階層のみを横切る場合、横切られなかったリングの親子関係は維持される。
    - **閉線切り替え円の仕様**:
        - 分断線入力中、最初の分割点を中心に画面固定サイズの円を表示する。
        - 円の中心は最初の分割点、半径はUI設定の固定値とする。
        - 2点目以降も円は表示し続ける。
        - 3点以上の分断線が入力された状態で円内をクリックした場合、分断線を閉じて閉線分割として扱う（円形の分割線が生成されるわけではない）。
        - 閉線に切り替えた時点で閉線パスを表示し、確定ボタンと同等のフローへ移行する。
    - **操作ガイド文**:
        - 分割ツール活性中は操作ガイド文を画面上に表示する。
        - 分断線の点はどこでも配置できることを明示する。
        - 3点以上でガイド円内をクリックすると閉線へ切り替わり、確定フローへ移行することを説明する。
    - **プロパティ継承先選択モーダル**:
        - 確定後、分割結果のプロパティ継承先を選ぶモーダルダイアログを表示する。
        - **配置**: 画面中央に表示する。将来の競合解決ダイアログと共通化できる構成とする。
        - **プレビュー**: モーダル中央に、分割結果のうち小さい方を囲む範囲を切り出した地図プレビューを表示する。
        - **ボタン配置**: プレビュー上に継承先を選ぶ2つのボタンを重ねて表示する。
            - ボタン選択時は対応する面情報をハイライト表示する。
            - 面情報を直接クリックしても選択状態が切り替わる。
        - **確定**: プレビュー下部に確定ボタンを配置し、選択がない場合は無効化する。
        - **非継承側のプロパティ**: 面情報追加時の入力フロー（UI・バリデーション・既定値の扱い）をそのまま再利用し、分割専用の入力ロジックを増やさない。
    - **競合解決プロセスへの遷移**: その後は「編集時の時間的・空間的整合性維持プロセス」の競合解決プロセスへ遷移する。
    - **操作の不可分性**: 分割は「既存面の形状更新」と「新規面の作成」を不可分の1操作として扱い、履歴記録・イベント通知・整合性検証は一括で完了させる。UIはこの複合操作に必要な入力を集めるための前段とする。

- **ナイフツール（閉線分割）**:
    - 分断線を閉じた場合は閉線分割として扱う（円形に限定しない、任意の閉じた形状）。
    - 閉線に基づいて面が2つに分割され、それぞれ新しい歴史の錨として記録される。
    - 確定時に「編集時の時間的・空間的整合性維持プロセス」をトリガーする。

#### 2.3.3.3 コンテキストメニュー機能

- アクティブ地物または現在の頂点選択セットを右クリックすることでコンテキストメニューを表示する。
- オブジェクト種類やツール種類によってメニュー内容が動的に変化する。
- **共通機能**:
    - 削除
    - プロパティ編集（アクティブ地物が存在する場合、または頂点選択コンテキストが一意に定まる場合）
- **共有頂点固有機能**:
    - 共有解除
- **面情報固有機能**:
    - 分割
    - 所属変更
    - 結合
- これらの操作も完了時に歴史の錨の変更として記録され、「編集時の時間的・空間的整合性維持プロセス」を経る。

## 2.4 注釈・説明機能

### 2.4.1 属性情報

- **基本属性**:
    - 名前: 地物の表示名称。
    - 年代: 開始年・終了年（歴史の錨の時間範囲として管理）。
- **色情報**:
    - 自動彩色が有効かどうか（自動配色許可フラグ）。
    - 塗り色（通常表示色）。
    - 選択時色（選択表示色）。
    - 境界色。

### 2.4.2 詳細文書

- マークダウン形式での詳細文書を各地物に添付可能。
- 画像の埋め込みをサポートする。
- 文書内のリンク機能（他のオブジェクトへの参照など）。
- 基本的なフォーマット（太字、斜体、リスト、表など）。

### 2.4.3 情報の表示

- **サイドバー**:
    - オブジェクトクリック時に表示する。
    - 基本情報（名前、年代、設定色）を即時表示する。
    - 編集機能を統合する（編集モード時）。
    - 「詳細表示」ボタンで完全な情報を「詳細パネル」として表示する。
- **詳細パネル**:
    - モーダルウィンドウ形式で表示する。
    - ドラッグ可能で複数同時表示可能。
    - すべての属性と詳細文書を表示する。
    - 編集機能を統合する（編集モード時）。

### 2.4.4 面スタイル仕様

#### 2.4.4.1 面スタイルの定義

各面情報の歴史の錨には以下のスタイル属性を持たせる。

- **通常表示色**: 通常時の塗りを表す色（透過度を含む色指定）。穴領域には適用しない。
- **選択表示色**: 当該面が選択されているときの塗り色。境界線の強調は既存仕様（シアンアウトライン）を継続する。
- **自動配色許可フラグ**: システムが色を自動更新してよい場合に真となるフラグ。フラグを変更しても色値はそのまま残る。
- **使用パレット**: 自動配色を行う際に参照するテーマ名。自動配色を無効化しても選択値は保持する。

#### 2.4.4.2 自動配色の挙動

1. ユーザーが自動配色を許可すると、現在の隣接関係に基づいて即座に色を決定する。
2. 許可状態が続く限り、隣接関係が更新されたタイミングのみで再配色が行われる。
    - 頂点の移動だけでは隣接関係が更新されないため、再配色しない。
3. 許可を解除するとシステムによる色更新は停止し、色は自動配色で作成された通りのまま保持される。ユーザーは手動で通常色・選択色を編集できる。
4. 再び許可した場合のみ、システムが色を更新する可能性が生まれる。
    - 許可しただけでは色は変わらない。隣接関係の変化があって初めて再配色される。

#### 2.4.4.3 自動配色の考え方

- **四色定理ベース**: 同一レイヤー内で共有境界を持つ面が同色にならないように色を割り当てる。
- **派生色の生成**: 四色で足りない場合は、選択パレットの基準色を基に明度・彩度をわずかにずらした派生色を決定的に生成して差別化する。
- **選択表示色の自動調整**: 選択表示色は通常表示色より明るく鮮やかになるように調整し、可読性を確保する。
- **飛び地の扱い**: 飛び地や包領は同じ面情報として扱い、面単位で一貫性のある色を用いる。
- **穴の扱い**: 穴（除外領域）は常に透過扱いとし、選択時にも塗りを乗せない。

#### 2.4.4.4 再配色トリガー

| イベント | 内容 |
| --- | --- |
| 共有境界の成立・解消 | 共有頂点や接続の新規作成/削除によって面同士が接するかどうかが変わったとき |
| 面の分割 | 面を複数に分け、新しい面が既存面に隣接する場合 |
| 面の結合・合邦 | 複数面が一つになり、隣接リストが再構成される場合 |

- 時間移動や単純な頂点移動など、隣接判定に影響しない操作では再配色を行わない。

#### 2.4.4.5 プロパティパネルUI（面スタイル）

面情報専用の「スタイル」セクションを設け、以下を配置する。

1. **自動配色を許可する**（チェックボックス）
    - ON: 「隣接関係が変わったとき自動で塗り分けを更新します」と案内を表示。
    - OFF: 「現在の色は維持され、システムは変更しません」と案内を表示。
2. **使用パレット**（ドロップダウン）
    - 自動配色がONの間のみ編集可能。
    - テーマ例: クラシック、パステル、ハイコントラストなど。
3. **通常表示色**（カラーピッカー＋透過スライダー）
    - 自動配色ON時は値の閲覧のみ可能。OFFで編集解放。
4. **選択表示色**（カラーピッカー＋透過スライダー）
    - 自動配色ON時は値の閲覧のみ可能。OFFで編集解放。

- 「地物または頂点が選択されていません」という既定メッセージは、選択状況に応じてスタイルセクションの有効/無効と連動させる。

#### 2.4.4.6 レイヤー表示とテーマ設定

- レイヤーは順序番号の小さいものから描画し、上位レイヤーほど後に描画される。これにより、面スタイルはレイヤーの積層順に従って重ね合わされる。
- レイヤーの不透明度と面スタイルの透過度は掛け合わせて最終的な見た目を決める。
- プロジェクト設定には以下を追加する:
    - **新規面の自動配色既定値**（ON/OFF）: 新しく追加した面情報に自動配色を有効にするかどうかの初期値。
    - **既定パレットテーマ**（プロジェクト全体で統一）: 新規面に適用されるデフォルトのパレット。
    - **カスタムパレット**（任意、複数色のセットを登録可能）: ユーザーが独自の配色セットを定義できる。

## 2.5 データ保存・読み込み

### 2.5.1 データ保存

- **ファイル形式**:
    - JSON形式のテキストファイルとする。
    - ファイル拡張子は `.json` とする。
    - 形式バージョン（version）をファイルに保存して互換性を管理する。
    - GeoJSON形式とは互換性を持たせない。

- **保存内容**:
    - すべての頂点、点情報、線情報、面情報。
    - 各オブジェクトの時間依存プロパティ（歴史の錨として）。
    - レイヤー構造と所属関係。
    - アプリケーション設定（スライダー範囲、世界名など）。

- **保存操作**:
    - メニューまたはショートカット（Ctrl+S）で保存する。
    - 「名前を付けて保存」機能も提供する。
    - 自動保存機能（設定された間隔で一時ファイルに保存）。

### 2.5.2 データ読み込み

- **読み込み操作**:
    - メニューまたはショートカット（Ctrl+O）で読み込む。
    - ドラッグ＆ドロップでのファイル読み込みもサポートする。

- **バージョン確認**:
    - ファイル内のバージョン情報を確認する。
    - システムが対応する形式バージョンと一致しない場合は、互換性の警告を表示する。

- **データ読み込み時の詳細な挙動**:
    - ファイルを読み込んだ時点で、データの形式バージョンの妥当性を検証する。
    - データ整合性チェックでは以下を確認し、不整合がある場合はユーザーが選択可能な自動修復オプションを提供する:
        - 存在しない頂点を参照している点・線・面情報
        - レイヤーの階層構造の不整合
        - 時間プロパティの矛盾（例: 開始日時が終了日時よりも未来に設定されている場合）
        - 歴史の錨間の空間的オーバーラップ（保存データに矛盾が含まれていた場合）

### 2.5.3 警告機能

- **未保存変更の警告**:
    - 保存されていない変更がある状態で以下の操作を行う際に警告表示する:
        - 新しいファイルを読み込む
        - アプリケーションを閉じる
        - 新しいプロジェクトを開始する
    - 警告ダイアログには「保存」「保存せず続行」「キャンセル」の選択肢を表示する。
- **データ互換性警告**:
    - 互換性のないバージョンのファイルを読み込む際に警告を表示する。
- **データ整合性チェック**:
    - 読み込み時にデータの整合性をチェックする（不正な参照や欠落した情報など）。
    - 問題がある場合は警告を表示し、修復オプションを提示する。

### 2.5.4 回復メカニズム

- **自動バックアップ**:
    - 設定可能な間隔で自動バックアップを作成する（デフォルトは5分ごと）。
    - バックアップファイルはアプリ本体の `/savebackup/` フォルダ内に、保存ファイル名ごとのサブフォルダを作成して格納する。
    - 各サブフォルダ内のバックアップは最大10世代まで保持し、超過分は古いものから削除する。
    - バックアップファイルは一意のタイムスタンプを持ち、これをファイル名に含める。

## 2.6 プロジェクト設定の編集

- ユーザーは、現在のプロジェクトに固有の設定を編集できるインターフェースを利用できる。

### 2.6.1 設定項目一覧

| 設定項目 | 説明 | 備考 |
| --- | --- | --- |
| プロジェクト名 | プロジェクトの表示名称 | 必須 |
| プロジェクト説明 | プロジェクトの概要テキスト | 任意 |

## 4. 内部実装補遺（データモデル）

### 4.1 基本データ構造

各データオブジェクトの構造は以下の通り。

- **Vertex（頂点）**
    - `id`: 文字列。一意の頂点ID（例: "vx-1234567890"）。
    - `x`: 数値。経度に対応。正距円筒図法における水平位置。経度生値設計（開発ガイド§6.2.16）により、180°超の地物では `±180` を超える値を保持する（例: `190` は「180°から東に10°」を意味する）。
    - `y`: 数値。緯度に対応（値域: -90.0 〜 90.0）。正距円筒図法における垂直位置。
    - 経度は生値のまま保存し、`normalize()` は保存・移動・描画の経路で呼ばない。表示用の折り返し（`-180〜180` への変換）は表示層でのみ行う。

- **TimePoint（時間点）**
    - `year`: 数値。年（必須）。
    - `month`?: 数値。月（オプション、1-12）。
    - `day`?: 数値。日（オプション、1-31）。

- **FeatureAnchor（歴史の錨実体）**
    - `id`: 文字列。歴史の錨の一意ID。
    - `timeRange`: オブジェクト。歴史の錨の有効期間。
        - `start`: TimePoint。開始時刻（必須）。
        - `end`?: TimePoint。終了時刻（排他的、省略時は無期限）。
        - 同一地物の錨列では `start` 昇順を維持し、時間重複を許可しない。
        - `end` が次の錨の `start` より前の場合、その期間は地物の非存在期間を表す。
    - `property`: オブジェクト。歴史の錨の属性情報。
        - `name`: 文字列。地物の名称。
        - `description`: 文字列。地物の説明。
        - `labelVisibility`?: オブジェクト。ラベル表示条件。
        - `style`?: オブジェクト。面情報のスタイル定義（2.4.4節参照）。面情報以外の地物では省略。
            - `fillColor`: 文字列。通常表示時の塗りつぶし色（RGBA）。
            - `selectedFillColor`: 文字列。選択表示時の塗りつぶし色（RGBA）。
            - `autoColor`: 真偽値。自動配色許可フラグ。真の場合、隣接関係の変化時にシステムが色を自動更新する。
            - `palette`: 文字列。自動配色に使用するパレットテーマ名。
        - `attributes`?: オブジェクト。拡張属性。
    - `shape`: オブジェクト。地物の形状定義。
        - 点情報の場合: `{ type: "Point", vertexId: "string" }`
        - 線情報の場合: `{ type: "LineString", vertexIds: ["string"] }`
        - 面情報の場合: `{ type: "Polygon", rings: [Ring] }`
    - `placement`: オブジェクト。所属と階層情報。
        - `layerId`: 文字列。所属レイヤーID。
        - `parentId`?: 文字列。上位領域ID。
        - `childIds`?: 文字列の配列。下位領域ID配列。

- **PropertyProjection（互換表示用の読み取り専用投影）**
    - `FeatureAnchor` から算出される表示専用ビュー。
    - 編集・保存・履歴記録の書き込み元として使用してはならない。

- **Ring（ポリゴンリング）** - ポリゴンを構成する要素
    - `id`: 文字列。リングの一意なID（例: "ring-123..."）。
    - `vertexIds`: 文字列の配列。頂点IDの配列（順序付き、最低3点）。
    - `ringType`: 文字列 (`'territory'` | `'hole'`)。リングの種類。`'territory'` は領土を、`'hole'` は穴を示す。
    - `parentId`: 文字列またはnull。このリングを直接含む外周リングのID。ポリゴンの最外周リングや、最上位の飛び地の外周リングの場合は `null`。

- **Feature（地物）**
    - `id`: 文字列。一意のID。
    - `featureType`: 文字列。地物の種類 ("Point", "Line", "Polygon")。
    - `anchors`: FeatureAnchorオブジェクトの配列。時系列にソートされた正準履歴。
    - `properties`: PropertyProjectionの配列。表示互換のための読み取り専用投影。**保存対象外**（`anchors` から都度算出される）。

- **Layer（レイヤー）**
    - `id`: 文字列。一意のID。
    - `name`: 文字列。レイヤー名。
    - `order`: 数値。レイヤー順序（小さいほど下位）。
    - `visible`: 真偽値。表示/非表示状態。
    - `opacity`: 数値。不透明度（0.0〜1.0）。
    - `description`?: 文字列。説明（省略可）。
    - レイヤーは可視性・描画順序・所属ツリーの管理のみを担い、色や線幅などのスタイル情報は保持しない。

- **SharedVertexGroup（共有頂点グループ）**
    - `id`: 文字列。グループの一意ID。
    - `vertexIds`: 文字列の配列。共有関係にある頂点IDのリスト（2つ以上）。
    - `representativeCoordinate`: Coordinate。代表座標（共有化確定時の座標）。
    - 共有解除時はグループからメンバーを除去し、メンバーが1つ以下になったグループは削除する。

- **TimelineMarker（タイムラインマーカー）**
    - `id`: 文字列。マーカーの一意ID。
    - `time`: TimePoint。マーカーが示す時刻。
    - `label`: 文字列。マーカーの表示名。
    - `description`?: 文字列。補足説明（省略可）。

## 5. 内部実装補遺（システムアーキテクチャ）

### 5.1 全体構成

システムは以下の4層のDDDアーキテクチャで構成される:

| 層 | 責務 |
| --- | --- |
| **ドメイン層** | 核となるビジネスロジックと規則 |
| **アプリケーション層** | ユーザー操作の処理と状態管理 |
| **インフラストラクチャ層** | データの永続化とレンダリング |
| **プレゼンテーション層** | ユーザーインターフェースと対話 |

### 5.2 ドメイン層

- **エンティティ**:
    - **Vertex**: 頂点の基本情報。
    - **Feature**: すべての地理オブジェクトの基底クラス。`id`, `featureType`, `anchors`（FeatureAnchorのリスト）を持つ。
    - **Layer**: レイヤー情報。

- **値オブジェクト**:
    - **Coordinate**: 座標値。
    - **TimePoint**: 時間情報。
    - **FeatureAnchor**: 特定の時間範囲における地物の完全状態スナップショット（形状、属性、所属、親子関係を含む）。
    - **PropertyProjection**: FeatureAnchor から導出される読み取り専用の表示投影。
    - **Ring**: ポリゴン形状を構成するリング。

- **ドメインサービス**:
    - **GeometryService**: 幾何学計算（距離、面積、交差、内外判定、ブーリアン演算など）。
    - **TimeService**: 時間管理と履歴処理。
    - **LayerService**: レイヤー間の関係管理。

- **リポジトリインターフェース**:
    - **WorldRepository**: データの保存と読み込みの抽象化。

### 5.3 アプリケーション層

#### 5.3.0 ユースケース一覧

- `EditFeatureUseCase`: 地理オブジェクトの編集処理のファサード。内部で歴史の錨の追加・変更・削除を行い、その結果として「編集時の時間的・空間的整合性維持プロセス」を呼び出す。
    - `AddFeatureUseCase`: 地物の追加。
    - `UpdateFeatureAnchorUseCase`: 歴史の錨の更新。
    - `DeleteFeatureUseCase`: 地物の削除。
    - `VertexEditUseCase`: 頂点の編集操作。
- `PolygonEditService`: 特定時間におけるポリゴン形状編集（リング操作など）のロジックを担当。`EditFeatureUseCase` から利用される。
- `NavigateTimeUseCase`: 時間移動の処理。
- `ManageLayersUseCase`: レイヤー管理の処理。
- `ResolveFeatureAnchorConflictsUseCase`: 「編集時の時間的・空間的整合性維持プロセス」ダイアログからのユーザーの選択を受け、具体的なデータ調整（ブーリアン演算の指示、プロパティ更新など）を実行する。詳細I/Oは5.3.1節を参照。

#### 5.3.1 歴史の錨編集の保存契約（I/O）

##### PrepareFeatureAnchorEditUseCase

- **入力**:
    - `featureId`: 編集対象地物ID
    - `editMode`: `"property_only"` または `"shape_and_property"`
    - `editTime`: TimePoint
    - `draftPatch`: 編集案（属性差分、期間、形状差分、所属差分）
        - `boundaryEdit`?: 錨境界編集案
            - `targetAnchorId`: 境界変更対象の錨ID
            - `newStart`?: TimePoint（新しい開始時刻）
            - `newEnd`?: TimePoint | null（新しい終了時刻。null は無期限）
- **出力**:
    - `draftId`: 保存前編集案ID
    - `status`: `"ready_to_commit"` または `"requires_resolution"`
    - `candidateAnchors`: 編集案適用後のアンカー列
    - `affectedTimeRange`: 影響を受ける時間範囲（競合検出対象）
    - `conflicts`: 競合一覧（`status="requires_resolution"` の場合のみ）

##### ResolveFeatureAnchorConflictsUseCase

- **入力**:
    - `draftId`: 保存前編集案ID
    - `resolutions`: 競合IDごとの解決方針
        - `preferFeatureId`: 優先する地物ID
- **出力**:
    - `resolvedAnchorsByFeature`: 競合解決後のアンカー列

##### CommitFeatureAnchorEditUseCase

- **入力**:
    - `draftId`: 保存前編集案ID
    - `resolvedAnchorsByFeature`: 競合解決後のアンカー列
- **出力**:
    - `updatedFeatures`: 確定後の地物集合
    - `historyEntryId`: 単一履歴操作ID

##### 契約制約

- `Prepare` 成功後にのみ `Resolve` / `Commit` を実行できる。
- `status="requires_resolution"` の場合、`Prepare` は競合ID・時刻・地物ペアを含む情報を返し、UIで全競合の解決方針指定を要求する。
- `status="requires_resolution"` の場合、すべての競合に方針が与えられるまで `Commit` を拒否し、方針不足時は再度解決要求を返して保存を拒否する。
- `Commit` は全地物を一括確定し、部分確定を禁止する。
- `boundaryEdit` が指定された場合、`Prepare` は同一地物内の錨列を開始時刻昇順・重複なしへ正規化した候補を生成しなければならない。
- 正規化後の錨列が0件になる場合、`Prepare` は失敗として扱う。
- `Resolve` は、優先地物の形状を維持し、非優先地物の同時刻有効錨へ差分演算（非優先 - 優先）を適用する。
- 非優先地物の競合錨が競合時刻より前から継続している場合、`Resolve` は同一地物錨を競合時刻で前後に自動分割し、後半錨へ差分形状を適用する。
- 競合時刻が錨開始時刻と一致する場合、`Resolve` は当該錨の形状を差分形状へ置換する。
- 差分結果が空の場合のみ、`Resolve` は非優先地物の存在期間打切りまたは錨削除を適用する。
- 解決適用で複数地物の履歴が更新された場合も、履歴は単一コマンドとして記録する。

#### 5.3.2 コマンド/クエリ/イベント

- CQRSの原則に基づき、操作（コマンド）、データ取得（クエリ）、状態変更通知（イベント）を分離する。

### 5.4 インフラストラクチャ層

- **データ永続化**:
    - `JSONWorldRepository`: JSONファイルを用いた `WorldRepository` の実装。
    - `FileSystem`: ファイルシステム操作の抽象化。
    - `JSONSerializer`: ドメインオブジェクトとJSON間の変換。

- **レンダリング**:
    - `SVGRenderer`: SVGを用いた地図描画。地図要素の生成・更新・削除を担う。
    - `ViewportManager`: 地図表示領域（ズーム、パン、中心経度）の管理。中心経度の取得・設定・相対シフトを提供する。

- **システムサービス**:
    - `Logger`: ログ記録。
    - `ConfigManager`: 設定管理。

### 5.5 プレゼンテーション層

Svelteコンポーネントと XState ステートマシンによる構成を採用する。各コンポーネントは表示と入力受付を担い、複雑な状態遷移はXStateマシンに委譲する。

- **Svelteコンポーネント**:
    - `MapCanvas`: メインマップ表示とユーザーインタラクション受付。SVGRendererと連携し、地図要素の描画を行う。
    - `TimelinePanel`: 時間軸スライダーと関連コントロール（再生、ステップ操作、速度選択）。
    - `Toolbar`: 編集モードやツールの切り替えボタン。アクティブなツールの状態を反映する。
    - `Sidebar`: レイヤー一覧、地物一覧、プロパティ表示・編集パネルを含むサイドバー。
    - `ConflictResolutionDialog`: 「編集時の時間的・空間的整合性維持プロセス」ダイアログのUI。
    - `LongitudeCompass`: 地図上部に表示される経度コンパス。目盛り・中心マーカー・ナッジボタン・ホイール/キーボード操作で中心経度を変更する。描画モード中は非表示。
    - `GridLabelsOverlay`: 地図ビューポート上に固定表示される緯度・経度ラベルのHTMLオーバーレイ。

- **XState ステートマシン**:
    - 各編集ツールの操作フロー（待機→選択→操作→確定）をステートマシンとして定義し、状態遷移を宣言的に管理する。
    - `toolMachine`: ツール共通の選択・待機・操作中などの状態遷移。各ツール固有のサブステートを持つ。
    - `conflictResolutionMachine`: 競合解決ダイアログの表示→選択→確定→適用の状態遷移。
    - `timelineMachine`: 再生・停止・ステップ操作・速度変更の状態遷移。
    - Svelteコンポーネントは `useMachine` を通じてステートマシンを購読し、現在の状態に応じたUIを描画する。ユーザー操作はイベントとしてマシンに送信される。

## 6. 用語集

| 用語 | 定義 |
| --- | --- |
| 点情報 | 単一地点を表す地理情報オブジェクト（都市、イベント地点など） |
| 線情報 | 複数の頂点で構成される線形地理情報オブジェクト（道路、国境など） |
| 面情報（ポリゴン） | 閉じた形状を持つ地理情報オブジェクト（国家、領域など）。リング構造で定義される |
| 頂点 | 地理オブジェクトの形状を定義する点 |
| リング | ポリゴンの形状（領土または穴）を定義する閉じた頂点の配列。識別子、頂点識別子列、リング種別、親リング識別子を持つ |
| 領土リング | ポリゴンの領土部分を定義するリング。ポリゴンの最外周や飛び地の境界線 |
| 穴リング | ポリゴン内部の除外領域を定義するリング |
| 飛び地 | 地理的に分離しているが、同じポリゴン地物に属する独立した外周リング |
| ネスト | リングが他のリングの内部に含まれる階層構造（例: 穴の中の飛び地） |
| 共有頂点 | 複数のオブジェクトで共有される頂点 |
| レイヤー | 地理情報を階層的に管理するための層 |
| 上位レイヤー | 対象レイヤーより階層が上のレイヤー |
| 下位レイヤー | 対象レイヤーより階層が下のレイヤー |
| 上位領域 | 対象の面情報が属する上位レイヤーの面情報 |
| 下位領域 | 対象の面情報に属する下位レイヤーの面情報 |
| 時間点 | 特定の時点（年/月/日） |
| プロパティ | 特定の時間から有効になる、地物の完全な状態スナップショット。形状、属性、レイヤー所属、親子関係などを含み、歴史の錨として機能する |
| 歴史の錨（アンカー） | ユーザーが特定の時間において明示的に定義した地物の状態。この「錨」から次の「錨」までは状態が継続する。過去の編集は未来の「錨」自体を自動変更しない |
| 競合 | ユーザーの編集操作の結果、時間的・空間的な不整合が発生する状態。例: ポリゴンの空間的オーバーラップ、イベントの前提条件不備など |
| 競合解決プロセス | 競合が検出された際に起動される、ユーザーが解決方針を選択するための専用ダイアログと関連処理 |
| 衝突判定 | 複数のポリゴン間の重なりを検出する処理 |
| エッジ滑り | 頂点が他のポリゴンのエッジに当たった際に沿って移動する動作（特定時間内の編集時） |
| データ整合性チェック | データの参照整合性、内部矛盾などを検出・修復する処理 |
| 独自歴 | ユーザーが設定した公転・自転周期などに基づく架空の暦法（将来実装予定） |
| 合体（結合） | 同じレイヤーに属する、下位領域を持たない複数の面情報を一つの面情報に統合する操作 |
| 合邦 | 下位領域すべてが別の上位領域に属することで、複数の面情報が一つの上位領域に統合される操作。合体（結合）とは違い、最下位の面情報の形状は維持される。形状自体の操作ではなく、あくまでも所属の操作 |
| アクティブ地物 | ユーザーが直接操作するために明示的に選択した地物 |
| 頂点選択コンテキスト | 頂点選択セットから一意に導かれる地物。プロパティ表示や補助ハイライトに使用する |

## 7. 将来実装したい機能

- **独自歴サポート**:
    - ユーザーが地球の暦とは異なる独自の暦法（1年の日数、1年の月数、各月の日数など）を設定できる。
    - タイムラインスライダーの目盛り、ラベル表示、および全ての時間関連の入力・表示を独自歴に合わせて自動調整する。
    - システム内部の時間情報（歴史の錨の時刻情報など）も独自歴に基づいて正規化・再計算される。
