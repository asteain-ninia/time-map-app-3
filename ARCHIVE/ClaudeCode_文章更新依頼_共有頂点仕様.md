# Claude Code への文章更新依頼: 共有頂点仕様

## 目的

共有頂点まわりの仕様文言を、実装と今後の設計方針に合わせて整理・更新してください。
文章の最終判断と表現調整は Claude Code 側に委ねます。

## 背景

- 現在、共有頂点化の禁止条件について議論中です。
- 以前は「同じ地物に属する頂点同士の共有頂点化を禁止」という方向で文言を入れています。
- ただし、その後の整理で「地物単位」より「形状コンポーネント単位」の方が仕様として自然、という結論に近づいています。
- 特に、同一地物の別リングは一律禁止ではなく、候補としては許容したい意向があります。
- 一方で、共有頂点化の結果として形状が壊れるケースは明確に禁止したいです。

## 文章更新の主眼

以下の意図が伝わるように、要件定義書の共有頂点仕様を再構成してください。

1. 禁止の主語は「地物」より「リング/線形などの形状コンポーネント」に寄せたい
2. 同一地物の別リングは一律禁止にしない
3. ただし、共有頂点化または共有頂点移動の結果、形状が壊れる場合は禁止する

## 現時点の設計意図

推したい不変条件は次です。

- 1つの共有頂点グループに、同一リングの頂点を2つ以上入れない
- 1つの共有頂点グループに、同一線形の頂点を2つ以上入れない
- 同一地物の別リングに属する頂点は、候補から一律除外しない
- ただし、共有頂点化または共有頂点移動の結果として、以下が起きるなら禁止する
  - ゼロ長エッジ
  - 連続重複頂点
  - 自己交差
  - 穴/飛び地の階層破綻
  - 同一レイヤー内の面重複

## 特に避けたい文脈

以下のように読める文章は避けたいです。

- 「同じ地物なら常に共有頂点化禁止」
- 「別リングなら常に共有頂点化可能」

やりたいのは、

- 同一リング/同一線形の多重点共有は抑止する
- 別リングは候補になりうる
- ただし最終的にはトポロジ検証に通った場合だけ確定する

という整理です。
