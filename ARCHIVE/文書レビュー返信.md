# 文書レビュー返信

レビューありがとうございます。全17件の指摘を精査し、対応しました。以下に各指摘への対応内容を記載します。

## A. 修正すべき問題

### A-1. セクション番号の重複 → 修正済み

- `2.6.1 アプリ設定` → `2.6.4 アプリ設定` にリネーム。
- 経緯: 直前の修正で `2.6.0` → `2.6.1` にリネームした際、既存の `2.6.1` との衝突を見落としたもの。
- **確認箇所**: 要件定義書.md の `### 2.6.4 アプリ設定`

### A-2. 保存形式例とデータモデルの不整合 → 修正済み

- 保存形式例（4.2節）の `attributes.styleOverrides` 配下の `fill`/`stroke`/`strokeWidth` を削除し、`property.style` 配下の `fillColor`/`selectedFillColor`/`autoColor`/`palette` に置換。
- `stroke`/`strokeWidth` は仕様本文（2.4.4節）で面ごとの境界線カスタマイズに言及がなく、選択時のシアンアウトライン強調が既存仕様として継続するため、データモデルには追加せず削除とした。
- 2つ目の錨（anchor-a-1200）にも `style` を追加。
- **確認箇所**: 要件定義書.md 4.2節のJSON例内 `"style": {` ブロック（2箇所）

### A-3. parentId "0" → 修正済み

- 保存形式例内の `"parentId": "0"` を `"parentId": null` に修正（2箇所）。
- データモデルの `parentId?` がオプショナルである定義と整合。
- **確認箇所**: 要件定義書.md 4.2節のJSON例内 `placement` ブロック

### A-4. 共有頂点グループのデータモデル → 追加済み

- 4.1節に `SharedVertexGroup（共有頂点グループ）` を追加:
  - `id`: グループの一意ID
  - `vertexIds`: 共有関係にある頂点IDのリスト（2つ以上）
  - `representativeCoordinate`: 代表座標
  - 共有解除時のグループ削除ルールも記載
- 4.2節の保存形式例に `sharedVertexGroups` 配列を追加。
- **確認箇所**: 要件定義書.md `SharedVertexGroup（共有頂点グループ）` の定義と、JSON例内の `"sharedVertexGroups"` 配列

### A-5. イベントマーカーのデータモデル → 追加済み

- 4.1節に `TimelineMarker（タイムラインマーカー）` を追加:
  - `id`: マーカーの一意ID
  - `time`: TimePoint（マーカーが示す時刻）
  - `label`: 表示名
  - `description?`: 補足説明（省略可）
- 4.2節の保存形式例に `timelineMarkers` 配列を追加。
- **確認箇所**: 要件定義書.md `TimelineMarker（タイムラインマーカー）` の定義と、JSON例内の `"timelineMarkers"` 配列

### A-6. 仕様本文内の英語識別子 → 修正済み

- 仕様本文（第1〜3章、第6〜7章）の範囲を検索し、以下を修正:
  - `RGBA` → 「透過度を含む色指定」（2.4.4.1節）
- `GeoJSON`（行788）および `WebWorker`（行983付近）は技術標準の固有名詞であり、内部識別子ではないため修正対象外とした。
- 第4〜5章（内部実装補遺）の英語識別子は第0章の方針で許可されている範囲。
- **確認箇所**: 要件定義書.md 2.4.4.1節の「通常表示色」定義

## B. 改善推奨

### B-1. 編集機能の分類位置 → 対応保留

- 指摘の妥当性は認めるが、セクション番号の大幅変更を伴う構造的リファクタリングとなるため、他の修正とは別のパスで対応予定。
- 現状の配置でも機能的な誤りや実装への影響はない。

### B-2. 用語集「合体（結合）」の定義不一致 → 修正済み

- 旧: 「隣接する**2つ**の**最下位**面情報を一つの面情報にまとめる操作」
- 新: 「同じレイヤーに属する、**下位領域を持たない複数の**面情報を一つの面情報に**統合する**操作」
- 仕様本文（2.1節の合体機能）の記述と完全に整合。
- **確認箇所**: 要件定義書.md 第6章用語集の「合体（結合）」行

### B-3. Svelte/XStateバージョン未指定 → 明記済み

- 技術方針.md を以下のように更新:
  - `UIフレームワーク: Svelte` → `UIフレームワーク: Svelte 5`
  - Runes API（`$state`, `$derived`, `$effect`）を使用する旨を明記。stores APIは使用しないと注記。
  - `状態管理: XState` → `状態管理: XState v5`
  - v4とのAPI差異に関する注意書きを追加。
- **確認箇所**: 技術方針.md セクション1.3および1.4

### B-4. Electron+Vite統合の構成指針 → 追記済み

- 技術方針.md セクション1.6を以下のように更新:
  - `ビルドツール: Vite` → `ビルドツール: Vite (electron-vite)`
  - electron-viteによる3エントリポイント（メインプロセス・プリロード・レンダラー）の統合管理を明記。
  - HMRはレンダラープロセスのみ対象であることを明記。
  - プロセス間通信（IPC）は `contextBridge.exposeInMainWorld` を用いるパターンを指定。
- **確認箇所**: 技術方針.md セクション1.6

### B-5. 座標系の定義 → 追記済み

- 要件定義書.md 4.1節のVertex定義を更新:
  - `x` = 経度（値域: -180.0 〜 180.0）
  - `y` = 緯度（値域: -90.0 〜 90.0）
  - 横方向無限スクロール時のラッピング方針（表示上は±180超、保存時は正規化値域内）を明記。
- **確認箇所**: 要件定義書.md 4.1節 `Vertex（頂点）` の定義

### B-6. メタデータ保存形式例の設定項目不足 → 追加済み

- 保存形式例の `metadata.settings` に以下を追加:
  - `gridColor`: グリッド線の色
  - `gridOpacity`: グリッド線の不透明度
  - `oblateness`: 扁平率
  - `labelAreaThreshold`: 面情報ラベル画面占有率閾値
  - `defaultAutoColor`: 新規面の自動配色既定値
  - `defaultPalette`: 既定パレットテーマ
  - `customPalettes`: カスタムパレット
- **確認箇所**: 要件定義書.md 4.2節のJSON例内 `"settings"` ブロック

### B-7. ファイル行数目安の整理 → 追記済み

- 開発ガイド.md 2.1節の「1ファイルあたり300行を目安に分割」に括弧書きで「Svelteコンポーネントは200行以下、2.5節参照」を追加。
- **確認箇所**: 開発ガイド.md 2.1節の行数目安記述
